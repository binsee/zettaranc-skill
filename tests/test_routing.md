# 路由逻辑测试用例

## 测试目标
验证 SKILL.md 中的路由逻辑能准确地将用户问题分流到对应模块。

## 测试用例

### 1. 股票/投资问题（原有逻辑）

#### 1.1 个股分析
**输入**：「600519.SH 怎么看？」
**预期路由**：股票/投资 → Step 1.5（个股问诊）
**验证点**：
- 是否进入多轮问诊流程
- 是否询问短线/长线、持有/想买、仓位占比

#### 1.2 持仓诊断
**输入**：「我买了茅台，成本1800，现在浮亏5%，该怎么办？」
**预期路由**：股票/投资 → Step 1.5（持仓诊断）
**验证点**：
- 是否询问买入原因、持仓天数、止损设置
- 是否给出明确的诊断结论

#### 1.3 技术指标
**输入**：「B1买点是什么意思？」
**预期路由**：股票/投资 → Step 2（研究）
**验证点**：
- 是否解释 B1 的定义和用法
- 是否引用 knowledge/trading-core.md

### 2. 人生/职业决策问题（新增逻辑）

#### 2.1 职业选择
**输入**：「我该选什么职业？」
**预期路由**：人生/职业决策 → 四个圈检查
**验证点**：
- 是否询问健康、能力、心性、时运四个维度
- 是否引用 knowledge/life-decision.md

#### 2.2 人生规划
**输入**：「我该怎么规划人生？」
**预期路由**：人生/职业决策 → 四个圈检查
**验证点**：
- 是否询问确定性、时间维度
- 是否引用 knowledge/life-decision.md

#### 2.3 转行决策
**输入**：「我想转行，但不知道该转什么？」
**预期路由**：人生/职业决策 → 职业发展四层模型
**验证点**：
- 是否询问当前职业层级、热爱什么、愿意付出多少时间
- 是否引用 knowledge/career-development.md

#### 2.4 赚第一桶金
**输入**：「我该怎么赚到第一桶金？」
**预期路由**：人生/职业决策 → 确定性思维
**验证点**：
- 是否询问热爱什么、能成为什么专家、愿意主动出击吗
- 是否引用 knowledge/life-decision.md 和 knowledge/career-development.md

#### 2.5 迷茫/困惑
**输入**：「我现在很迷茫，不知道该怎么办？」
**预期路由**：人生/职业决策 → 四个圈检查
**验证点**：
- 是否从四个圈角度分析问题
- 是否给出明确的方向性建议

### 3. 创业/商业判断问题（新增逻辑）

#### 3.1 创业方向
**输入**：「我该创业做什么？」
**预期路由**：创业/商业判断 → 时代主线检查
**验证点**：
- 是否询问看得懂、有确定性、在周期什么位置
- 是否引用 knowledge/business-judgment.md

#### 3.2 赛道选择
**输入**：「这个赛道值得投资吗？」
**预期路由**：创业/商业判断 → 赛道选择三原则
**验证点**：
- 是否询问看得懂、有确定性、在周期什么位置
- 是否引用 knowledge/business-judgment.md

#### 3.3 行业分析
**输入**：「怎么判断一个行业有没有前途？」
**预期路由**：创业/商业判断 → 时代主线判断
**验证点**：
- 是否讲解时代主线、价格重新定价、掐头去尾抓中间
- 是否引用 knowledge/business-judgment.md

#### 3.4 年轻人投资
**输入**：「我是年轻人，该怎么投资？」
**预期路由**：创业/商业判断 → 新兴消费文化
**验证点**：
- 是否询问消费什么产品、同龄人讨论什么、哪些品牌有话题性
- 是否引用 knowledge/business-judgment.md

### 4. 模糊/边界问题

#### 4.1 模糊关键词
**输入**：「我不知道该怎么办？」
**预期路由**：需要根据上下文判断
**验证点**：
- 如果上下文是股票问题 → 股票/投资
- 如果上下文是人生问题 → 人生/职业决策
- 如果上下文是创业问题 → 创业/商业判断

#### 4.2 多领域混合
**输入**：「我想投资股票，但不知道该怎么学习？」
**预期路由**：股票/投资 + 人生/职业决策
**验证点**：
- 是否同时涉及投资和学习能力提升
- 是否引用 trading-core.md 和 career-development.md

#### 4.3 上下文切换
**输入**：
- 第一句：「我最近在研究股票。」
- 第二句：「你觉得我该转行做投资吗？」
**预期路由**：从股票/投资切换到人生/职业决策
**验证点**：
- 是否检测到话题切换
- 是否重新进入四个圈检查流程

---

## 测试执行方法

### 方法1：手动测试
1. 在 Claude Code 中激活 zettaranc-perspective skill
2. 逐一输入测试用例
3. 观察路由是否符合预期
4. 记录测试结果

### 方法2：自动化测试
1. 使用 Claude Code 的测试工具
2. 编写测试脚本，自动输入测试用例
3. 自动验证路由结果
4. 生成测试报告
